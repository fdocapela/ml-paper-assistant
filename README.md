# 🧠 ML Paper Analysis Assistant

Multi-agent system for analytical question-answering over 5 landmark ML papers,
powered by Gemini 2.0 Flash, ChromaDB, and FastAPI.

---

## Architecture

```
User (HTTP)
    │
    ▼
┌─────────────────────────────────────────────────────────┐
│                  FastAPI REST API                       │
│          /threads   /threads/{id}/messages              │
└────────────────────────┬────────────────────────────────┘
                         │
                         ▼
┌─────────────────────────────────────────────────────────┐
│           OrchestratorAgent  (Gemini 2.0 Flash)         │
│  receives question + thread history → calls tools →     │
│  consolidates final answer                              │
└──────────────┬──────────────────────────┬───────────────┘
               │ function calling         │ function calling
               ▼                          ▼
┌──────────────────────┐      ┌───────────────────────────┐
│      RAGAgent        │      │       AnalystAgent        │
│  • search_documents  │      │  • compare_papers         │
│  • extract_section   │      │  • summarize              │
│                      │      │  • rank_papers            │
└──────────┬───────────┘      └──────────────┬────────────┘
           │                                 │
           ▼                                 ▼
┌─────────────────────────────────────────────────────────┐
│               ChromaDB  (Vector Store)                  │
│   5 ML papers · chunked · embedded · indexed            │
└─────────────────────────────────────────────────────────┘
           │
           ▼
┌─────────────────────────────────────────────────────────┐
│               SQLite  (Thread Memory)                   │
│   Per-thread conversation history · SQLAlchemy async    │
└─────────────────────────────────────────────────────────┘
```

## Prerequisites

You only need two things installed on your machine:

- **Docker Desktop** — [docs.docker.com/get-docker](https://docs.docker.com/get-docker/)
- **Git** — [git-scm.com](https://git-scm.com/)
- **Python 3.11+** — only needed to run `make cache` once

No other local dependencies. Everything else runs inside Docker.

## Setup — Two Commands

### 1. Configure your environment

Get a free Google AI Studio key at **aistudio.google.com**, then:

```bash
cp .env.example .env
```

Open `.env` and set your key:

```
GOOGLE_API_KEY=AIzaSyYOUR_REAL_KEY_HERE
```

> ⚠️ Never commit `.env` to Git — it is already listed in `.gitignore`.

### 2. Add the PDF files

Place the 5 paper PDFs inside the `papers/` folder with these exact names:

```
papers/attention_is_all_you_need.pdf
papers/bert.pdf
papers/rag.pdf
papers/react.pdf
papers/toolformer.pdf
```

### 3. Download packages to local cache (run once, needs internet)

Because Docker containers on some machines cannot reach the internet during
the build phase, all packages are pre-downloaded to a local folder on your
host machine and then copied into the Docker image:

```bash
make cache
```

This downloads ~100 Python wheel files into `pip_cache/`. It only needs to
run once. The folder is excluded from Git (see `.gitignore`).

### 4. Build and start everything

```bash
make setup
```

This single command:
1. Builds the Docker image (installs packages from `pip_cache/` — no internet needed)
2. Starts the `app` and `chromadb` containers
3. Waits for ChromaDB to be healthy
4. Ingests the 5 papers into ChromaDB (chunking + embedding)

When complete:
```
✅ Setup complete!
   API:     http://localhost:8000
   Swagger: http://localhost:8000/docs
```

## API Endpoints

| Method | Route | Description |
|---|---|---|
| `POST` | `/threads` | Create a new conversation thread |
| `POST` | `/threads/{thread_id}/messages` | Send a question to the orchestrator |
| `GET` | `/threads/{thread_id}/messages` | Get full message history |
| `GET` | `/threads` | List all threads |
| `GET` | `/health` | Health check |

### Example

```bash
# Create a thread
curl -X POST http://localhost:8000/threads

# Ask a question (replace <id> with the UUID returned above)
curl -X POST http://localhost:8000/threads/<id>/messages \
  -H "Content-Type: application/json" \
  -d '{"content": "Compare ReAct and Toolformer for tool use in LLMs."}'
```

## Daily Commands

| Command | What it does |
|---|---|
| `make cache` | Download pip packages (run once) |
| `make setup` | Build + start + ingest papers |
| `make run` | Run the 5 evaluation questions |
| `make test` | Run all automated tests |
| `make logs` | Follow live application logs |
| `make down` | Stop and remove containers |
| `make ingest` | Re-ingest papers (e.g. after config change) |
| `make shell` | Open a shell inside the app container |
| `docker compose up -d` | Start again after `make down` |

## Technical Decisions

| Decision | Choice | Rationale |
|---|---|---|
| **Vector Store** | ChromaDB 0.5.x | Runs as a Docker container, async HTTP client |
| **Embeddings** | `text-embedding-004` | Current Google embedding model |
| **LLM** | Gemini 2.0 Flash via `google-genai` SDK | Latest SDK, native async, function calling |
| **Chunking** | 800 tokens, 150 overlap | Captures full arguments, preserves context across boundaries |
| **Agent framework** | Direct SDK — no LangGraph/CrewAI | Simple 2-agent architecture, full control |
| **Persistence** | SQLite + SQLAlchemy async | Zero-dependency database for thread history |
| **PDF handling** | Papers committed to repository | Zero runtime downloads, works offline |
| **Docker networking** | pip_cache pre-downloaded on host | Bypasses Docker SSL/network restrictions |

## Known Limitations

1. No streaming — full answer returned after all tool calls complete
2. PDF parsing — `pypdf` may struggle with complex mathematical notation
3. Rate limits — Google AI Studio free tier has RPM limits
4. Thread history — truncated to last 10 messages per request
5. No authentication — API is open, not production-ready without auth middleware
